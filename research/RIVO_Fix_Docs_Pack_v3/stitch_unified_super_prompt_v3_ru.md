# RIVO Fix Configurator — Stitch Unified Super‑Prompt v3 (RU)
Дата: 2026‑02‑21

## Роль и цель
Ты — ведущий UI/UX + Product designer (enterprise), специализация: инженерные конфигураторы (2D/CAD + параметры + проверки + экспорт).  
Собери **полный набор экранов** и **дизайн‑систему**, чтобы пользователь собирал типовую конструкцию за **3–5 минут** без ручных расчётов.  
Интерфейс **на русском**, тексты ошибок и подсказок — тоже.

## Визуальный стиль (референс‑микс)
- **Minimal + glass/neotactile**: мягкие поверхности, лёгкие тени, аккуратный blur (8–24px), округления 16–24px.
- **Engineering clarity**: тонкие линии, сетка, читаемые размерные подписи, строгая иерархия.
- **Motion**: micro‑interactions 150–280ms, easing `cubic-bezier(0.2,0.8,0.2,1)`; пересчёт — skeleton + progress.

## Layout (desktop-first 1440+)
- **Left**: Stepper + группы параметров (accordion).
- **Center**: главный **фронтальный** вид (2D/CAD‑readable).
- **Right**: **боковой** вид + sticky summary (валидность/ошибки/экспорт/цена).
- Всегда доступно: переключение XY/XZ/YZ, reset view, измерения.

## IA (обязательные экраны)
1. Тип конструкции
2. Габариты (W/H/D + обшивка + основание)
3. Оборудование
4. Проверки (нагрузка/устойчивость/геометрия)
5. CAD‑режим
6. Итог (спецификация + валидность)
7. Экспорт (PDF/DXF/IFC)
8. (Доп.) Профиль/Проекты: список, шаринг ссылкой, версии

## Поведение конфигуратора (обязательные UX‑правила)
- Любая недоступность опции: **почему** + **что сделать** (кнопки: «перейти к параметру», «исправить автоматически»).
- При изменении параметров: показывай состояние «Пересчёт…», затем обновляй модель/проверки/спецификацию.
- Валидация: `Valid / Warning / Error`. Ошибки блокируют экспорт; предупреждения — разрешают с пометкой.
- Undo/Redo; Jump‑to‑step; сравнение версии (v2+).

## Компоненты (обязательные)
- Stepper (с группами и статусами)
- Numeric input с единицами (мм) + min/max + stepper + ввод с клавиатуры
- Dropdown / Segmented control
- Tooltip “Почему недоступно?”
- Banner ошибок/предупреждений (с CTA)
- Table BOM/спецификация (SKU/артикул/кол-во/длина/цена/итого)
- Export modal (PDF/DXF/IFC) + прогресс + toast “Экспорт готов”
- Viewer toolbar (оси/размеры/слои/сетка/контраст/снимок)

## CAD‑режим (строго)
- Белый фон, чёрные линии, осевые штрих‑пунктир.
- Размерная сетка, отметка 0.000, обязательные размеры: общая W/H/D, межосевые, высоты монтажных элементов.
- Слои: контур, оси, размеры, оборудование, примечания.

## Состояния
- loading (skeleton) / empty / error / partially‑valid (warnings)
- disabled options с объяснением и ссылкой на причину

## Accessibility
- WCAG 2.2 A/AA: фокус‑индикатор 2px, targets ≥ 24px, контраст ≥ 4.5:1 для текста
- Клавиатурная навигация для всех контролов, ARIA‑лейблы для сложных элементов.

## Deliverables в Stitch
Сгенерируй:
- Полные экраны (все пункты IA)
- Полную дизайн‑систему (tokens + компоненты + states)
- Кликабельный прототип 3 сценариев:
  1) «С нуля» → экспорт PDF+DXF  
  2) «Ошибка/исправление» (несовместимость)  
  3) «Редактирование по ссылке» (изменить параметр → перепроверка → экспорт)

## Контент‑правила
- Короткие заголовки, подсказки по делу.
- Технические термины: «обшивка», «основание», «инсталляция», «проверка устойчивости».
- Все сообщения локализуемые (готовь `code` + `message`).

## Важное
Не придумывай бизнес‑логику изделия. UI должен быть готов принимать ответы движка:
- `validity`, `messages[]`, `suggestedFixes[]`, `bom[]`, `price`.
