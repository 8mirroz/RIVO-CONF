# Stitch Prompt — Усиление Constraint Explainability (RU)

Добавь системный слой объяснений ограничений и авто‑фиксов.

## Формат сообщения (модель)
Каждая проблема — объект:
- `severity`: error | warning | info
- `code`: STRING
- `title`: коротко
- `message`: причина человеческим языком
- `relatedParams`: список параметров (ключи)
- `impact`: что ломает (экспорт/устойчивость/цена)
- `actions[]`: 
  - `label`
  - `type`: jump_to_param | auto_fix | open_help
  - `target`: ключ шага/параметра
  - `payload`: значения для автофикса (опционально)

## UI паттерны
- Sticky “Проверки”/“Статус” справа: счётчик ошибок/предупреждений.
- Внутри шагов: inline‑errors под конкретным параметром.
- Для disabled options: tooltip “Почему недоступно?” + ссылка “исправить”.

## Авто‑фикс
- Кнопка “Исправить автоматически” всегда показывает *что изменится* (diff‑preview).
- После фикса — toast: “Исправлено: {параметр} → {значение}”.

## Копирайтинг (шаблоны)
- “Недоступно, потому что …”
- “Чтобы продолжить: …”
- “Рекомендуемое исправление: …”

## Аналитика
- Логируй события: constraint_shown, fix_applied, fix_rejected.
